Alt Text Extension — Test Page

Open this file in Chrome with the extension loaded. Each case below tests a specific classification. Hit Re-scan Page in the popup after the page loads.

Missing Alt Text — expect RED outline + "!" badge

Case 1 — no alt attribute at all
<img src="...">
Expected: RED outline, listed in popup as missing
Case 2 — no alt, inside an <a> tag
<a><img src="..."></a>
Expected: RED outline
Case 3 — no alt, inside <figure> with <figcaption>
Expected: RED outline — caption ≠ alt text

Has Alt Text — expect GREEN outline

Case 4 — meaningful alt text
<img src="..." alt="A green test image">
Expected: GREEN outline
Case 5 — alt text with surrounding prose

Our team presenting at the 2024 conference.

Expected: GREEN outline

Decorative Images — expect ORANGE outline

Case 6 — alt="" (explicitly empty — HTML spec decorative)
Expected: ORANGE outline
Case 7 — role="presentation"
Expected: ORANGE outline
Case 8 — aria-hidden="true"
Expected: ORANGE outline

Lazy-loaded Image — expect RED after src swap

Case 9 — lazy load via data-src pattern

Currently shows a 1×1 transparent placeholder. The real src is in data-src.

Expected: detected via data-src; goes RED after button click triggers src swap

Edge Cases

Case 10 — 1×1 tracking pixel (no alt)
Expected: SKIPPED entirely (too small to be content)
Case 11 — data: URI image with no alt
