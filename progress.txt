Context-Aware Alt Text Support - Project Progress
===================================================
Team Rocket: Rayaan Maredia, Kaiqi Yang, Simeon Belay, Gaston Schweitzer

COMPLETED
---------

Week 1 - Image Detection & Extension Scaffold
  [x] Chrome extension structure (Manifest V3)
  [x] Content script scans all <img> elements on any page
  [x] Classifies images: missing alt text, has alt text, decorative
  [x] Red outline + warning badge (!) on images missing alt text
  [x] Green outline on images with alt text
  [x] Orange outline on decorative images
  [x] Decorative detection heuristics (role=presentation, aria-hidden, empty alt, tiny images)
  [x] Badge count on extension icon showing number of missing alt texts
  [x] Popup with accessibility score (0-100), stats grid, and image list
  [x] MutationObserver watches for dynamically loaded images
  [x] Re-scan button in popup

Week 2 - AI Caption Generation (TAMU API)
  [x] Settings page for API key, server selection (all TAMU system servers), and model choice
  [x] API integration module using TAMU OpenAI-compatible chat completions endpoint
  [x] Vision support: sends images as base64 with surrounding page context
  [x] System prompt for generating concise, accessible alt text
  [x] Decorative image detection via AI (responds "DECORATIVE")
  [x] Click warning badge (!) on any image to generate alt text for that image
  [x] "Generate All Alt Text" button in popup for batch processing
  [x] Generated alt text is injected into the DOM dynamically
  [x] Visual feedback: loading spinner, success checkmark, error state
  [x] API key stored locally in chrome.storage (never shared)
